# iOS 快捷指令配置指南 (最新版)

本指南指导你如何在 iOS 设备上创建快捷指令，将收据文字准确发送到 `BillScan Worker`。

## 1. 核心配置信息

- **目标 URL**: `https://billscan-worker.zeromarx2.workers.dev`
- **请求方法**: `POST`
- **数据格式**: `JSON`

## 2. 快捷指令步骤详解

按照以下步骤在“快捷指令” App 中配置：

### 第一步：获取收据内容
1. **选择照片** 或 **拍摄照片**。
2. **从图像中提取文本** (使用系统内置 OCR)。
   - *提示：将此输出命名为“识别出的文本”。*

### 第二步：输入付款人
1. **要求输入**：提示文字为“付款人姓名？”，默认值可设为“自己”。
   - *提示：将此输出命名为“付款人”。*

### 第三步：发送请求 (关键步骤，防转义错误)
**不要手动拼接 JSON 字符串**，因为 OCR 文本中的换行符和引号会破坏格式。请按以下方式配置“获取 URL 内容”操作：

1. **URL**: 输入 `https://billscan-worker.zeromarx2.workers.dev`
2. **方法**: 选择 `POST`
3. **头部 (Headers)**:
   - 添加新项目：键为 `Content-Type`，值为 `application/json`
4. **正文 (Body)**: 选择 **JSON**
   - 点击 **添加新项目** -> **文本**：
     - 键 (Key): `text`
     - 值 (Value): 选择变量 **“识别出的文本”**
   - 点击 **添加新项目** -> **文本**：
     - 键 (Key): `paidBy`
     - 值 (Value): 选择变量 **“付款人”**

> **为什么这样配置？**
> iOS 快捷指令的“正文: JSON”模式会自动处理文本中的转义字符（如换行符 `\n`、引号 `"` 等）。即使 OCR 识别出的内容非常乱，它也能确保发送给服务器的是一个合法的 JSON 格式。

## 3. 常见问题排查

- **提示 "Invalid JSON"**: 通常是因为在快捷指令中手动用文本框拼接了 JSON 导致。请务必使用上述“添加新项目”的列表模式。
- **识别不全**: 确保拍摄收据时光线充足，文字清晰。
- **500 错误**: 如果返回 500，可能是 AI 暂时无法解析该特定格式。你可以尝试再次运行。
